Componentes

Procesos que hacen funcionar el sistema

El sistema corre en cuatro procesos Python simultáneos más el EA de MT5. Click para detalle.

🧠
trader_brain.py
Orquestador principal. Loop a 2s. Llama a los LLM, escribe órdenes, gestiona estado.
⚙️
ClaudeBrainEA (MQL5)
Expert Advisor en MetaTrader 5. Ejecuta órdenes en el broker con baja latencia.
📺
brain_flow.py
Servidor web del dashboard. Streaming de eventos vía SSE. Controles manuales.
🐕
brain_watchdog.py
Supervisor cada 60s. Reinicia el brain si muere, restaura el chart, limpia ghost states.
🌐
Cloudflare Tunnel
Túnel nombrado que expone el dashboard en bot.compteslab.net. Soporta SSE sin buffering.
🔗
TradingView CDP bridge
tv.js — Node.js CLI que habla con TradingView Desktop por Chrome DevTools Protocol.
Filosofía de diseño

Cinco principios que guían cada línea de código

1 Razonar como un trader, no como un bot

Sin umbrales fijos arbitrarios. Cada decisión es una hipótesis evaluada en contexto completo. El LLM solo cambia de opinión por eventos estructurales (BOS, sweep, rebote confirmado), nunca por micro-movimientos dentro del ruido del ATR.

2 Honestidad estructural

Si una zona no tiene evidencia real en los datos, el Indicator no la marca. Saber que no hay red es información tan valiosa como saber dónde está. Nunca se inventan soportes para hacer parecer que el trade está protegido.

3 Capas redundantes de seguridad

DD hard stop al 3.5% en el EA + projected DD al 85% en el brain + BE automático + staircase TPs en el broker + watchdog reinicio + cluster suppression. Si una capa falla, hay otras tres.

4 Reflejo vs Decisión

El FAST engine es reflejo (rápido, sin juicio complejo). El Executor es decisión (juicio, dimensionamiento). Esta separación evita que el sistema actúe sin razonar cuando debería pensar, y no se paralice cuando hacen falta reacciones inmediatas.

5 Diagnóstico permanente

Cada bug detectado se resuelve con fix estructural permanente, nunca con parches temporales. Los problemas conocidos (chart-swap, race conditions, sniper huérfanos, clusters de zonas) tienen todos su mecanismo dedicado en el código.
